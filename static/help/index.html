Tutorial System

The main window of the tutorial system is split into two sections, the description and feedback. Its top frame describes the problem and provides helpful hints and examples. The bottom frame consists of three sections (Test Selection, Test Output, and Analysis).

The bottom frame has the following sections:

  • The 'Test Selection' section displays the tests which will be run when you check your code (F5). Tests define what you must do to pass the problem, and may consist of expected input and output. If you fail a test it will turn red, while passing a test will make it turn green. If your test looks like it will not work for similar inputs the test will turn orange. If the tests cannot be run, this will count as a failure, and they will turn red.
  • The 'Test Output' is displayed by selecting a test. If the test has failed, an explanation of the failure will be displayed between the dashed lines at the top of this section. If you used print to display any information, this will be shown beneath this in blue. You can change which test this section displays by clicking on a different test in the 'Test Selection' section above.
  • The 'Analysis' section provides tips and hints to help you complete the problem. Once your code is correct, you shouldn't see anything here. We require you to complete some problems in a specific way (for example, by using a for loop. If so, this requirement will be shown in the 'Analysis' section.

You must use the provided IDLE edit window in which to write your answers. Always make sure to submit your answer once it passes by pressing (F6). We will try to do this automatically wherever possible, but it's up to you to ensure that your problems are submitted on time. You can tell if a problem has been submitted by looking for "Submitted: Yes" just above the 'Test Selection' section.

The main window has the following menus:

  • The 'Problems' menu allows you to pick a particular tutorial problem to work on.
  • The 'Online' menu provides an interface to your online information. Allowing you to login, logout and view submisions.
  • Then 'Tools' menu which allows you to visualise your code and open the interpreter
  • The 'Preferences' menu allows you to configure MyPyTutor.
  • The 'MyPyTutor Feedback' menu allows you to provide us with feedback on any given tutorial problem and on the system overall.

Once the first problem of a session is chosen, an IDLE edit window will appear. You are to write your answer to the given problem in this window. The menus for this window are mostly the same as the IDLE edit window. The main differences are given below.

  • Because your solutions are saved and loaded from your answers folder the File menu is a little different. The 'Revert' option replaces your code with the default code from the problem, allowing you to start again. The 'Save' option saves your code to the answers folder. When you choose a problem, your saved code is automatically loaded and so there is no 'Open' option.
  • The 'Check' menu is similar to the 'Run' menu of IDLE but instead of running the code in the interpreter, it checks to see your answer is correct.
  • The 'Online' menu behaves in the same way as the Online menu in the main window.

Window Resizing

You can resize the main MyPyTutor window in the same way as any other window.

The Preferences Menu

  • The 'Configure Tutorials Folder' item allows you to choose the folder containing all the tutorial problems. The menu items in the 'Tutorials' menu are determined by the chosen folder.
  • The 'Configure Answers Folder' item allows you to choose the folder in which your (partial) answers are saved to and loaded from.

MyPyTutor can work with multiple tutorials (for example, from different courses). You are able to set up new tutorials and also remove them. You can change between tutorials and set the default tutorial (the one used on startup).

The Online Menu

If the tutorial is configured to provide an online interface and you are registered (i.e. have a username and password for MyPyTutor) then you can use this menu to interact with your online information.

A description of the options are give below.

  • Login: this connects you with online system using your UQ username and password.
  • Logout: this disconnects you from the online system.
  • Synchronise Solutions: this allows you to synchronize your solutions with the MyPyTutor network. This provides an online mechanism for saving your code that complements your local save via the Save option in the File menu of the editor. When closing and opening MyPyTutor, and if connected the internet, your solutions will automatically be synced.
  • Submit Answer: submit your answer (for marks). Your answer must be checked (and correct) before you can submit your answer.
  • Show Submissions: display all the tutorial problems showing which ones you have submitted on time ('OK') and which ones you have submitted late ('LATE'). It also shows your current percentage (late submissions are not counted).

Common Errors

  • Name 'xxxxx' is not defined -> You need to define the variable, function or class eg. xxxxx = (Some Value), def xxxxx(): or class xxxxx():
  • 'xxxx' != Expected Result -> Your result is not equal to what is expected (you may not be printing or returning the correct answer).
  • False is not True -> This is a variation of the above failure; in other words, you're not returning the right value. Look at the test description to understand in more detail why the test is failing.
  • xxxxx() takes 0 positional arguments but 2 were given -> Your function (or a function you are calling) takes 0 arguments when it should be taking 2. You should add arguments to your function eg. xxxxx(arg1, arg2) and/or make sure that you are providing the correct number of arguments to any function you are calling.
  • None != xxxxx -> You are not returning any value or you are returning None, make sure you are using return and not print
  • Make sure your code also works for similar inputs -> Make sure that you write your code so that it passes all tests (and all variations on a test), not just a single test.
  • ['xxxx','yyyy'] != ('xxxx','yyyy') -> Make sure that you are returning a tuple '()' and not a list '[]'
  • Error in code: test not run (possible infinite loop) -> Check your while loops and for loops, will they even finish looping?
